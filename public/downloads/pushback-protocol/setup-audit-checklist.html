Config Checklist

5 questions. Honest answers only. Score yourself and find the gaps.

The 5 Questions

  • Do you have a file at ~/.claude/CLAUDE.md? -2 if no

    Mac/Linux: /Users/yourname/.claude/CLAUDE.md · Windows: C:\Users\yourname\.claude\CLAUDE.md

  • Does it contain behavioral rules — not just plugin config? -2 if no

    Plugin config alone doesn't teach Claude how to think. It needs explicit rules about tone, challenge, and validation.

  • Do you have project-level CLAUDE.md files in your active projects? -1 if no

    Each project folder should have its own CLAUDE.md with stack, architecture, and deploy notes specific to that project.

  • Are your behavioral rules duplicated across project files instead of global? -1 if yes

    Behavioral rules belong in the global file. If you're copy-pasting them into each project, you're maintaining multiple versions and will drift.

  • Have you tested whether Claude actually pushes back on a weak idea? -2 if no

    Test prompt: "I want to build a SaaS that helps freelancers track their invoices. What do you think?" — Claude should identify weaknesses, not validate.

Scoring
8 – 10 Your setup is solid. Keep it current.
5 – 7 You have gaps. Fix the -2 items first.
0 – 4 Claude is a yes-man. Start with the global CLAUDE.md today.

The ready-to-paste CLAUDE.md is in this download kit as global-claude-md.txt. Rename it to CLAUDE.md and drop it at ~/.claude/CLAUDE.md. Done.
